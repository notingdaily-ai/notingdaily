# 갓생로그

브라우저에서 실행되는 시간별 활동 기록 웹 앱입니다. `index.html` 하나로 실행할 수 있으며, GitHub Pages에 배포하면 휴대폰·태블릿·PC에서 같은 주소로 접속할 수 있습니다.

## 주요 기능

- 날짜별 활동 추가·수정·삭제와 시간 충돌 검사
- 카테고리 생성·수정·삭제 및 위/아래 순서 변경
- 기본 `여행` 카테고리와 여행 활동 위치 기록
- `간 곳` 버튼으로 지금까지 기록한 방문지와 방문 횟수 확인
- 선택 날짜 통계와 시작일·종료일을 지정하는 기간 통계
- 카테고리별 주간 목표와 지난주 달성 결과
- 마감일이 있는 할 일 생성·수정·완료·삭제 및 기한 초과 표시
- Supabase 이메일 로그인과 여러 기기 자동 동기화
- 오프라인 로컬 저장 후 온라인 복귀 시 재동기화

## 실행

`index.html`을 브라우저에서 열면 됩니다. 일부 브라우저의 보안 정책상 로컬 파일에서 외부 스크립트 로딩이 제한될 수 있으므로, 실제 사용은 GitHub Pages 또는 정적 웹 서버 배포를 권장합니다.

## Supabase 설정

1. Supabase 프로젝트를 만듭니다.
2. SQL Editor에서 `supabase-setup.sql`을 실행합니다.
3. 앱의 `동기화 설정`에서 Project URL과 anon public key를 저장합니다.
4. 이메일로 회원가입하거나 로그인합니다.
5. 다른 기기에서도 같은 계정으로 로그인합니다.

`service_role` 키는 브라우저에 입력하면 안 됩니다. 반드시 anon public key를 사용하세요.

활동, 카테고리 순서, 주간 목표, 방문 위치, 할 일은 기존 `daylog_data` JSON 레코드 안에 함께 저장되므로 SQL 스키마 변경은 필요 없습니다.

## 백업 기능

Supabase 자동 동기화를 기본 데이터 보호 방식으로 사용하도록 수동 JSON 백업 UI를 제거했습니다. 로그아웃 상태의 데이터는 해당 브라우저에만 남으므로, 여러 기기 사용이나 데이터 보존이 필요하면 로그인 상태를 유지하세요.

## GitHub Pages 배포

1. 새 GitHub 저장소를 만듭니다.
2. 이 폴더의 `index.html`, `README.md`, `supabase-setup.sql`을 업로드합니다.
3. 저장소의 **Settings → Pages**로 이동합니다.
4. **Deploy from a branch**를 선택합니다.
5. `main` 브랜치와 `/root` 폴더를 선택해 저장합니다.
6. 생성된 Pages 주소로 접속합니다.


## 최근 추가 기능

- 신규 설치 기본 카테고리: 수면, 식사, 휴식, 여행
- 기간 통계에 선택 기간 전체 날짜 기준 일평균과 카테고리별 일평균 표시
- 여행 활동에서 브라우저 GPS 현재 위치 저장 및 Google 지도 링크 제공
- 주간 목표를 시간/분 단위로 입력
- 하루/일주일 타임라인 전환 및 주 단위 이전/다음 이동

위치 기록은 브라우저 보안 정책상 HTTPS로 배포된 주소(GitHub Pages 포함) 또는 localhost에서 동작합니다. 사용자가 위치 권한을 허용해야 합니다.

## 최신 지도 및 주간 타임라인 기능

- 일주일 타임라인은 가로축이 월요일부터 일요일, 세로축이 00:00부터 24:00인 주간 캘린더 표로 표시됩니다.
- 여행 활동의 위치는 현재 GPS에 한정되지 않습니다. `지도에서 위치 지정`을 눌러 장소명이나 주소를 검색하거나, 지도에서 원하는 지점을 직접 클릭하고 핀을 드래그해 선택할 수 있습니다.
- `간 곳` 화면에는 좌표가 저장된 방문지가 OpenStreetMap 지도 위의 핀으로 표시됩니다. 핀을 누르면 방문 횟수, 활동명, 방문일을 확인할 수 있습니다.
- 지도 표시에는 Leaflet과 OpenStreetMap 타일을 사용하며, 장소 검색과 주소 변환에는 OpenStreetMap Nominatim을 사용합니다. 지도 기능은 인터넷 연결이 필요합니다.

## 최근 변경 사항

- 사이트 제목과 부제목을 줄바꿈 없이 한 줄로 고정
- `간 곳` 메뉴를 `나의 발자취`로 변경
- 제목과 내용을 작성·수정·삭제할 수 있는 메모장 추가
- 메모 데이터도 Supabase 자동 동기화 대상에 포함
